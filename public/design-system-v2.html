Design System

Extracted from supafolio-landing.html and supafolio-security.html. This is the source of truth for the authenticated experience. All tokens, components, and patterns documented here must be used verbatim — do not introduce new values outside this system.

Tokens — CSS custom properties

All tokens live in :root. Never hardcode a hex value that has a token equivalent. The full token set is copied into the <style> block at the top of this document — use it as your copy-paste source.

Use .display-* and .body-* utility classes in production code. The .spec-* classes are documentation-only. display-num applies font-variant-numeric: tabular-nums automatically — use for all monetary values.

Weight rules: DM Sans ships at 300 (light), 400 (regular), 500 (medium). Use 300 for body copy and card text, 400 for UI labels and metadata, 500 for headings-in-sans, button labels, and emphasis. Never use 600 or 700.

Z-index ladder
TokenValueUsage
--z-base0In-flow content

--z-sticky10Sticky table headers, sticky page sub-nav

Never hardcode z-index values. Use only tokens from this ladder. Components that create stacking contexts (transforms, opacity < 1) must not be ancestors of elements that need to escape to a higher z-level.

Elevation: App frames use box-shadow: 0 24px 64px rgba(0,0,0,0.08), 0 4px 16px rgba(0,0,0,0.04). Floating toasts use 0 8px 24px rgba(0,0,0,0.2). Button hover uses 0 8px 24px var(--accent-glow). All other surfaces are flat — no shadow.

States — disabled, loading, dark

Disabled: opacity: 0.45; cursor: not-allowed; pointer-events: none on all variants. On dark bg, use btn-white (solid) or btn-ghost-dark (outline). Never btn-primary or btn-secondary directly on dark.

Rules

All buttons use border-radius: 100px (pill shape). Never use rectangular buttons. Icon buttons use the same padding with a square implicit dimension. Disabled state: opacity: 0.45; cursor: not-allowed; pointer-events: none. Loading state: replace label with an inline spinner — never hide the button. On dark backgrounds, use btn-white only — never btn-primary.

// Shared chart configuration tokens
CartesianGrid: strokeDasharray="3 3" stroke={paperBorder} vertical={false}
XAxis / YAxis: tick={{ fill: ink3, fontSize: 11 }} stroke="transparent"
Area stroke: var(--chart-1) | var(--danger), strokeWidth: 2, strokeLinecap: "round"
Area fill: linearGradient — stopColor same as stroke, stopOpacity 0.12 → 0
Dot: r={4}, activeDot={{ r: 6 }}
Line stroke: same as Area stroke, strokeWidth: 2, dot: false, activeDot={{ r: 4 }}
Pie stroke: paperBorder, strokeWidth: 1
Tooltip: content={<CustomTooltip />} — never use default Recharts tooltip
Animation: isAnimationActive={!prefersReducedMotion}, animationDuration={750}
ResponsiveContainer: width="100%" height={chartHeight}
  chartHeight: 300px desktop, 200px mobile (set via useEffect + matchMedia)
Y-axis: domain=calculateNiceDomain(), ticks=calculateNiceTicks()
  Compact format: ≥1M → "1.2M", ≥1K → "450K", else raw
